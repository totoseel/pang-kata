# PANG 개발 계획 — Phase별 목표

각 Phase는 **실제로 브라우저에서 실행되고 플레이 가능한 상태**를 목표로 한다.
Phase가 끝날 때마다 고객이 직접 플레이하며 피드백을 주는 방식으로 진행한다.

---

## Phase 1 — 게임 캔버스 & 플레이어 이동

> "화면이 열리고, 캐릭터가 움직인다"

### 구현 목표
- 고정 크기 게임 캔버스 렌더링
- 플레이어 캐릭터 좌우 이동 (키보드 ←→)
- 게임 루프(requestAnimationFrame) 동작
- 플레이어가 화면 밖으로 나가지 않도록 경계 처리

### 고객 확인 포인트
- 브라우저에서 게임 화면이 잘 뜨는가?
- 캐릭터가 좌우로 부드럽게 움직이는가?
- 화면 끝에서 캐릭터가 막히는가?

---

## Phase 2 — 풍선 물리 (바운스)

> "풍선이 튀어다닌다"

### 구현 목표
- Large 풍선 1개 생성 및 화면 내 바운스 (벽·바닥·천장 반사)
- 풍선 크기별 바운스 높이·이동 속도 차등 적용
- 풍선이 화면 밖으로 이탈하지 않도록 경계 처리

### 고객 확인 포인트
- 풍선이 벽과 바닥에서 자연스럽게 튕기는가?
- 풍선이 화면 밖으로 사라지거나 뚫고 나가지 않는가?
- 바운스 움직임이 실제 팡 게임과 유사한 느낌인가?

---

## Phase 3 — 작살 발사 & 풍선 분열

> "작살을 쏘면 풍선이 터지고 나뉜다"

### 구현 목표
- 스페이스바로 작살 발사 (위 방향, 1발 제한)
- 작살이 천장에 닿거나 풍선에 맞으면 소멸
- 풍선 분열 로직: Large → Medium×2 → Small×4 → Tiny×8 → 소멸
- 분열 시 좌우로 퍼지는 방향 처리

### 고객 확인 포인트
- 작살 발사 타이밍과 속도가 자연스러운가?
- 풍선이 맞았을 때 좌우로 자연스럽게 나뉘는가?
- Tiny 풍선까지 터뜨리면 완전히 사라지는가?
- 작살 1발 제한이 잘 동작하는가? (화면에 작살이 있는 동안 추가 발사가 안 되는가?)

---

## Phase 4 — 충돌 판정 & 목숨 시스템

> "풍선에 닿으면 목숨이 줄고, 0이 되면 끝난다"

### 구현 목표
- 플레이어-풍선 충돌 감지 및 목숨 1개 차감
- 초기 목숨 3개, HUD에 목숨 수 표시
- 목숨 차감 후 잠시 무적 시간(깜빡임 효과)
- 목숨 0 → 게임 오버 화면 표시

### 고객 확인 포인트
- 풍선에 닿았을 때 목숨이 정확히 줄어드는가?
- 피격 후 무적 시간 동안 다시 맞지 않는가?
- 목숨이 0이 되면 게임 오버 화면이 뜨는가?
- 충돌 판정이 너무 넓거나 좁지 않은가?

---

## Phase 5 — 스테이지 클리어 & 점수

> "모든 풍선을 없애면 클리어되고 점수가 쌓인다"

### 구현 목표
- 모든 풍선 제거 시 "STAGE CLEAR" 표시 후 다음 스테이지 전환
- 풍선 크기별 점수 적용 (Large 100점 / Medium 200점 / Small 300점 / Tiny 400점)
- 스테이지 클리어 보너스 10,000점
- 화면 상단에 현재 점수 표시

### 고객 확인 포인트
- 풍선을 터뜨릴 때마다 점수가 올라가는가?
- 크기가 작을수록 더 높은 점수가 쌓이는가?
- 스테이지 클리어 후 자연스럽게 다음 스테이지로 넘어가는가?

---

## Phase 6 — Mission 1 스테이지 구성 (블록 포함)

> "Mission 1의 3개 스테이지를 처음부터 끝까지 플레이할 수 있다"

### 구현 목표
- Stage 1 (Morning): Large 1개, 장애물 없음
- Stage 2 (Afternoon): Large 1개 + Medium 1개, 파괴 가능 블록 배치
- Stage 3 (Night): Large 2개, 파괴 가능 + 파괴 불가 블록 혼합
- 블록과 풍선·작살 간 충돌 처리
- Stage 3 클리어 시 Mission 1 완료 화면

### 고객 확인 포인트
- 3개 스테이지를 순서대로 플레이할 수 있는가?
- 블록이 풍선의 이동 경로를 실제로 바꾸는가?
- 파괴 가능 블록은 작살에 맞아 없어지는가?
- 파괴 불가 블록은 작살·풍선 모두 막는가?
- Stage 1 → 2 → 3으로 갈수록 체감 난이도가 올라가는가?

---

## Phase 7 — 아이템 드롭 시스템

> "풍선을 터뜨리면 아이템이 나오고, 집으면 효과가 발동된다"

### 구현 목표
- 풍선 파괴 시 확률적 아이템 드롭
- 아이템 6종 구현
  - 시계: 모든 풍선 일시 정지
  - 별: 화면 내 모든 풍선 즉시 제거
  - 모래시계: 풍선 이동 속도 감소
  - 실드: 피격 1회 무효화
  - 다이너마이트: 모든 풍선을 Tiny 크기로 축소
  - 보너스 과일: 추가 점수 획득
- 아이템이 바닥에 닿으면 소멸

### 고객 확인 포인트
- 아이템 효과가 즉시 체감되는가? (시계: 풍선이 멈추는가, 별: 전부 사라지는가 등)
- 아이템 드롭 빈도가 너무 많거나 적지 않은가?
- 아이템을 밟지 않으면 효과가 발동되지 않는가?
- 실드가 있을 때 한 번 맞아도 목숨이 줄지 않는가?

---

## Phase 8 — 무기 업그레이드 시스템

> "강력한 무기를 얻으면 플레이 방식이 달라진다"

### 구현 목표
- 무기 아이템 3종 구현
  - 더블 작살: 좌우 2발 동시 발사
  - 파워 작살: 천장에 고정되어 일정 시간 유지
  - 발칸 미사일: 연사 가능, 블록 파괴 불가
- 무기 아이템 획득 시 기존 무기 교체
- 현재 장착 무기 HUD에 표시
- 무기 지속 시간 종료 후 기본 작살로 복귀

### 고객 확인 포인트
- 더블 작살이 좌우 2발을 동시에 쏘는가?
- 파워 작살이 천장에 닿은 후 일정 시간 유지되다 사라지는가?
- 발칸 미사일로 연사 시 블록이 파괴되지 않는가?
- 무기가 바뀌었을 때 HUD에 즉시 반영되는가?
- 강한 무기를 얻었을 때 확실히 유리해지는 느낌인가?

---

## Phase 9 — 메인화면 & 화면 전환 흐름

> "처음 켰을 때부터 게임 오버까지 자연스러운 흐름으로 이어진다"

### 구현 목표
- 메인화면: 타이틀 "PANG", GAME START / HIGH SCORE 메뉴
- 화면 전환 흐름 완성
  - 메인 → 게임 시작 → 스테이지 진행 → 게임 오버 → 메인
- 게임 오버 화면에서 재시작 가능
- 하이스코어 저장 및 표시 (localStorage)

### 고객 확인 포인트
- 메인화면이 자연스럽게 보이는가?
- GAME START를 누르면 곧바로 게임이 시작되는가?
- 게임 오버 후 메인으로 돌아가는 흐름이 매끄러운가?
- 하이스코어가 새로고침 후에도 유지되는가?

---

## Phase 10 — UI 완성도 & 전체 polish

> "완성된 게임처럼 보이고, 플레이하기 좋다"

### 구현 목표
- 전체 UI 정돈: 폰트, 색상, 레이아웃
- 스테이지 배경 이미지 적용 (후지산)
- 풍선·플레이어·블록 스프라이트 또는 명확한 시각 표현
- 점수 팝업 (풍선 터질 때 점수 숫자 표시)
- 스테이지 시작 전 카운트다운 또는 스테이지 번호 표시
- 전반적인 애니메이션·효과 다듬기

### 고객 확인 포인트
- 전체적으로 "게임답다"는 느낌이 드는가?
- 풍선을 터뜨릴 때 시각적 피드백이 있는가?
- 스테이지 배경이 분위기를 만들어 주는가?
- 처음 접하는 사람도 직관적으로 조작 방법을 알 수 있는가?
